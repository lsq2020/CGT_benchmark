一、产品核心定位与基础配置
1. 平台基础定位
全称：🧬 ProteinDesign Bench 题目收集与管理平台
定位：蛋白质设计领域 AI Agent 评测基准，面向全国科研团队的开放出题、审核、全生命周期管理平台
核心目标：构建标准化、可溯源的蛋白质设计 AI 评测题库，目标收录题目总数 >1000 道
2. 难度层级与领域分类（预设方案）
（1）5 个难度层级定义
表格
难度等级	名称	核心场景与能力要求
L1	基础知识检索	考察对蛋白质基础数据、公共数据库、经典概念的检索与整理能力（如查询特定蛋白的 Uniprot 信息、PDB 结构、保守结构域）
L2	单任务序列 / 结构分析	考察对单一维度数据的分析能力（如基于模板的同源建模、序列保守位点分析、二级结构预测）
L3	简单功能设计与预测	考察针对单一功能目标的设计 / 预测能力（如单点突变设计、小分子结合位点预测、酶催化位点预测）
L4	多步骤理性设计	考察多步骤、多约束条件下的理性设计能力（如酶底物特异性改造、多靶点结合蛋白设计、蛋白稳定性提升设计）
L5	复杂系统级设计	考察系统级、多组件协同的设计能力（如合成生物学通路蛋白组设计、多蛋白复合物界面设计、动态构象调控设计）
（2）6 个蛋白质设计领域大类
表格
领域大类	细分方向示例
蛋白质序列设计	从头序列生成、基于模板的序列优化、密码子优化、突变文库设计
蛋白质结构预测与建模	单体结构预测、蛋白 - 蛋白复合物建模、蛋白 - 配体复合物建模、构象动态分析
蛋白质功能改造与设计	酶工程改造、结合蛋白设计、信号蛋白功能调控、蛋白稳定性 / 可溶性优化
蛋白质 - 配体相互作用设计	小分子结合口袋设计、抑制剂 / 激动剂设计、蛋白 - 核酸相互作用设计、结合亲和力优化
多蛋白复合物与系统设计	蛋白 - 蛋白界面设计、多亚基复合物组装设计、合成生物学通路蛋白组设计
AI 辅助蛋白质设计方法	设计算法评估基准、模型性能测试问题、设计流程自动化方案
二、页面结构与核心功能模块
整体设置 4 个核心标签页：「提交题目」「已提交」「已审核题目」「题库总览」，各模块细化如下：
（一）标签页 1：提交题目（出题表单页）
适配蛋白质设计场景，表单字段如下（带 * 为必填项）：
页面顶部：展示平台全称、副标题、实时数据看板（已收集总数、目标数、已审核数）
题目信息模块
题目标题 *（单行文本）：如 “人源 COX-2 选择性抑制剂结合口袋关键设计位点检索”
难度等级 *（下拉单选）：L1-L5
领域大类 *（下拉单选）：6 个预设方向
领域小类（单行文本）：如 “酶底物特异性改造”
题目正文 *（多行文本）：完整描述题目、子问题、作答要求、溯源要求（需明确标注文献 / 数据库条目）
采分点 Rubric 模块
规则：3-5 个采分点，总分必须为 10 分；每个采分点含 “可判断的描述”+“自定义分值”
示例：[4 分] 正确列出≥3 个关键残基及其 Uniprot ID；[4 分] 准确说明每个残基的结合作用；[2 分] 标注≥1 篇支撑文献 DOI
参考答案模块
多行文本，L1-L2 必填，L3-L5 强烈建议填写，需与采分点一一对应
来源与出题人模块
题目来源 *（下拉单选）：原创、文献改编、教材改编、数据库条目改编
来源详情（单行文本）：改编类必填，如 “doi:10.1038/s41586-024-xxxxxx；PDB ID: 6COX”
出题人姓名 *、单位 *、邮箱 *（自动校验格式）
提交按钮：校验通过后可点击，提交成功跳转至「已提交」页
（二）标签页 2：已提交（待审核题目列表）
筛选 / 排序：按难度、领域、提交时间、出题人筛选，按时间 / 难度排序
题目卡片：展示难度 / 领域 / 来源标签、题目标题 + 正文摘要、提交时间 + 出题人，支持折叠展开查看采分点 / 答案
操作权限（仅出题人可见自有题目）：编辑此题（仅待审核可编辑）、撤回提交、查看审核状态
（三）标签页 3：已审核题目（审核管理核心页）
1. 审核列表总览
顶部看板：待审核数、已审核数、需修改数
筛选：按审核状态（全部 / 已审核 / 需修改）、难度、领域、审核时间、出题人筛选
题目卡片：展示难度 / 领域 / 审核状态标签（已审核 - 绿色、需修改 - 黄色）、题目标题、提交 / 审核时间、出题人 / 审核人
操作权限：
出题人：查看详情、查看审核意见（需修改题目）、编辑重提（需修改题目）
审核员：额外展示「进入审核」按钮，可修改题目状态
2. 单题审核详情页（仅审核员可操作）
左侧：题目完整信息只读预览（全量字段、采分点、答案、出题人）
右侧审核面板：
审核状态选择 *（单选）：已审核、需修改
审核意见模块：
已审核：可选填写审核评语
需修改：必填项，含「预设修改原因复选框（可多选：题目描述不清晰、采分点设置违规、参考答案错误 / 不完整、溯源信息缺失、领域不符、其他）」+「自定义修改意见多行文本框（必填，详细说明修改要求）」
审核人信息：自动填充当前审核账号姓名 / 单位，不可修改
提交审核结果：同步更新题目状态
（四）标签页 4：题库总览（新增，支持导出）
展示范围：仅展示「已审核」通过的题目
功能：
检索：支持按题目标题、领域、难度、出题人关键词搜索
筛选：按领域、难度、审核时间筛选
题目展示：卡片形式展示题目标题、难度 / 领域标签、出题人 / 审核时间，支持点击查看完整题目信息
导出功能：支持将筛选后的题目批量导出为 Excel/JSON/Markdown 格式（仅审核员可导出全量，出题人可导出公开已审核题目）
三、权限体系与角色定义
表格
角色	核心权限
出题人	提交题目、编辑 / 撤回自有待审核题目、查看所有已审核题目、查看自有题目的审核状态与意见、编辑重提需修改的自有题目、导出公开已审核题目
审核员	所有出题人权限 + 查看全量待审核 / 已审核题目、进入单题审核页修改题目状态、填写审核意见、导出全量已审核题目
四、后端对接与服务器存储（暂定方案）
1. 后端对接（暂定）
协议：默认采用 HTTP/HTTPS RESTful API 对接实验室后端服务
核心接口：用户登录鉴权、题目提交 / 查询 / 编辑 / 撤回、审核结果提交、题库数据导出
后续可根据实验室实际后端架构调整对接方案
2. 服务器存储（暂定）
存储位置：所有数据存储在实验室内部服务器指定目录（后续可自定义路径）
存储格式：
结构化数据（题目信息、审核状态、用户信息）：默认存入 MySQL 关系型数据库
非结构化数据：按题目 ID 分文件夹存储在服务器文件目录
备份：支持定期自动备份全量数据到服务器备份目录
安全：所有操作留存日志，支持内网访问限制、登录鉴权
